# Dahuang Human Tone

## 这是什么

一个去除中文文本 AI 味的 skill。

它做一件事：**把"像模型在表演写作"的文本，拉回"像具体人在当前场景下说话"**。删的是模板壳、路标词、升华收尾、假互动——不是堆替换词。保的是信息、事实、立场、术语、作者锋利——不是抹平成"干净中性说明书"。

去味不是无脑删词，是改组织方式。

姐妹 skill：[`dahuang-ai-tone`](https://github.com/realchendahuang/dahuang-ai-tone)——把文本加成 GPT/Claude/Gemini/豆包的味，或者检测 AI 味在哪、像哪个风格。

## 触发场景

用户说这些走本 skill：`去AI味` / `去味` / `改自然点` / `不像AI写的` / `太像AI了` / `人味` / `humanize` / `去模型腔` / `删套话` / `改成人话`。

## 去味工作流

收到要去的文本，按这个顺序走（不要跳步）：

### 1. 判场景

先读一遍，判它在什么场景里跑：

| 场景 | 特征 | 去味力度 |
|---|---|---|
| 公开文（公号/博客/随笔/专栏） | 有作者声音，要有人味 | 标准 + 注入灵魂 |
| 社媒（小红书/微博/朋友圈） | 短、口语、有网感 | 标准，小心别改成说明书 |
| 技术文（文档/教程/说明） | 中性即正确，不需要个性 | 最小，只删协作腔和路标词 |
| 学术/正式报告 | 严谨客观，术语保留 | 最小，只删宣传腔和假互动 |
| 小说/故事 | 要声音、要画面、要节奏 | 标准 + 注入灵魂，小心套话 |
| 聊天回复（chat） | 简短直接 | 最小，只删谄媚腔和假互动 |

判不准时只问一个澄清问题，不要连发三问。

### 2. 划保护线

动手前先划清楚什么不能动：

- **事实、数字、日期、引用** —— 一字不改
- **专有名词、术语、命令、代码** —— 不动
- **引文、对话、直接引述** —— 不动
- **作者明确的有意选择**（比如刻意用的某个词、某个比喻）—— 尊重，别自作主张改掉
- **场景合法用词**（技术语境里的"接住请求""收口""落盘"是行话，不是 AI 味）—— 见 `references/guardrails.md` 的宾语判别
- **中英混排中的英文段落** —— 英文部分不按中文口癖规则处理，只改中文部分（详见 `references/guardrails.md`）
- **代码注释和 docstring** —— 技术注释以清楚为第一优先，不注人味、不删正式语气

### 3. 扫味

按五个层扫，哪层命中多就重点处理哪层。不要一上来读全部 reference——按需读：

| 层 | 找什么 | 详见 |
|---|---|---|
| 词汇层 | 口癖词、路标词、黑话、调试腔、自媒体腔、伪学术词 | `references/patterns.md` 第一节 |
| 句式层 | 二分对照、机械顺序、本质拔高、否定式排比、三段式、关联句堆叠 | `references/patterns.md` 第二节 |
| 结构层 | 段落同构、冒号模板、大纲骨架、升华收尾、意义盖章、元论述宣告 | `references/patterns.md` 第三节 |
| 格式层 | 破折号过载、粗体滥用、内联标题列表、emoji 装饰、弯引号 | `references/patterns.md` 第四节 |
| 交流层 | 协作痕迹、知识截止、谄媚腔、假互动结尾、二人称过密 | `references/patterns.md` 第五节 |

长文（≥1000 字）或公号文/论说文/评论文，加扫深层句式破绽（分析师讲解语姿、戏剧化揭露、极值判断、伪学术政论腔、标签讽刺、接力比喻、分析包装三件套、各种"壳"）——见 `references/deep-tells.md`。

任何长度的文本，都过一遍形状审计（段落均匀度、冒号模板密度、"很"字判断密度、句长标准差、排比过密）——见 `references/shape-audit.md`。

### 4. 删壳保信息

删的是壳，不是信息。最小改写原则：

- **能局部改就不整段重写**。删掉模板句式，把信息塞回叙述流（用逗号、分号、插入语接住），不扩散成整段重写。
- **删二分对照壳**："不是 A 而是 B" → 把 B 直接说清楚，A 如果多余就删，如果有信息就换普通句式说。
- **删路标词**："值得注意的是""接下来我们" → 删掉，直接说那件事。
- **删升华收尾**："愿我们都能……""未来可期" → 收在具体事实或克制判断。
- **删假互动**："你觉得呢？""你有没有类似经历？" → 删掉，除非用户要求互动。
- **改段落同构**：连续三段"观点句 + 解释 + 段尾总结"→ 打断其中一段，换节奏。
- **过载的破折号、粗体、emoji** → 删掉，只在真正需要强调时留。

**不要做机械同义词替换**。把"深入探讨"换成"讨论"不够——要把整句的结构改掉，让"深入探讨"这个位置不再需要存在。

### 5. 两遍

第一遍：扫味 → 删壳 → 出初稿。

第二遍（不可跳）：自问**"这段为什么一看就是 AI 写的？"**，列出来还剩什么味，再改一遍，出终稿。

如果初稿已经没什么味了，第二遍可以只做机械自检（见第 7 步），不用大改。

### 6. 注入人味（仅公开文/社媒/小说/随笔）

删壳只是工作的一半。无菌、没声音的写作和机器生成的一样明显。如果是需要作者声音的场景，去味之后还要注人味——有观点、变节奏、承认复杂性、适当用第一人称、对感受要具体。见 `references/soul.md`。

技术文/学术文不需要这步——中性就是正确的人味。

### 7. 出厂自检

终稿交付前做一次 grep 式扫描，逐条对终稿全文跑下面这些模式，命中就要处理（除非有刻意保留的理由）：

**口癖残留扫描**（命中即标记，逐条决定删/留）：

- `不是.{0,10}而是`、`并非.{0,10}而是`、`不只是.{0,10}更是`
- `值得注意的是`、`值得一提`、`需要指出的是`、`接下来我们`、`下面我们`
- `先说结论`、`一句话总结`、`省流版`、`说白了就是`、`说人话就是`
- `希望这能帮到你`、`你觉得呢？`、`你有没有`、`是不是`
- `愿我们`、`未来可期`、`让我们一起`、`综上所述`、`总而言之`
- `给您` / `您……`（正式商务/客服场景除外，其余 `您 → 你`）

**形状审计扫描**（量化信号，详见 `references/shape-audit.md`）：

- 破折号 `——` 全文出现次数；如果 ≥3 或单段多次，能换成逗号/句号的换
- 段落长度均匀度 + 冒号模板密度 + 句长标准差 + 段尾收束密度 + 排比过密 + 二人称过密 + 同构段落 + emoji 标题——**直接按 `shape-audit.md` 的审计项和阈值跑一遍**，不在 SKILL.md 这里手抄一遍清单。

**自检原则**：扫到的是"残留可疑点"，不是"必改命"。每个命中再回到第 4 步的最小改写原则决定是否动手——能局部删壳就别整段重写。

### 8. 平庸化回滚

去味完读一遍。如果发现：

- 段落变薄、信息密度下降
- 句子变短变均匀（原来是长短交错的）
- 作者的锋利/偏见/口吻被抹平了
- 读起来"太稳、太中、太像说明书"

→ **回滚到去味前的版本，只保留必要的修补**。去味是删壳，不是删人。宁可保留一点偏见与锋利，也不要改成通用的"干净中性文"。

## 核心规则速查

1. **删壳不删信息** —— 删的是模板句式和路标词，不是事实和观点
2. **保风格删路标** —— 保留作者的锋利和口吻，只删 AI 表演腔
3. **最小改写** —— 能局部改就不整段重写，把信息塞回叙述流
4. **不做同义词替换** —— 换词不够，要改结构让那个位置不再需要存在
5. **两遍不可跳** —— 初稿 → 自审还剩什么味 → 终稿
6. **别误伤行话** —— 技术语境的"接住/收口/落盘"是行话，不是 AI 味，看宾语判
7. **别过度删** —— 去味不是去个性，平庸化就回滚
8. **集群才定罪** —— 单个口癖不定 AI 味，一堆口癖叠一起才是

## 交付格式

默认**只输出去味后的终稿**，不输出流程、清单、自检记录。

用户要求解释/对比/诊断时，才加附注：

```md
## 去味后

{终稿}

## 删了什么

1. {壳/路标/收尾——具体改了什么组织方式}
2. {结构变化——具体改了什么}

## 还剩什么味（诚实说）

{如果还有残留，指出来；如果没有，说没有}
```

诊断模式（用户说"先别改，先标问题"/"只做诊断"）：输出 1-5 个问题点，每条标明 `问题在哪` / `是什么味` / `建议怎么改`，不输出改写版。

## 不接的活

遇到这些请求，**主动澄清或拒绝，不要硬接**：

- **英文/其他语种文本去味**：本 skill 只针对中文。英文请用 [`blader/humanizer`](https://github.com/blader/humanizer) 等英文 humanizer。澄清一次后等用户确认是否仍然要跑（会按中文口癖风格处理，效果有限）。
- **"帮我通过 AI 检测器"**：明确说不承诺、也不建议信检测结果（见下方"边界"）。检测器本身不准，骗它没有意义。
- **"帮我改事实 / 改数据 / 改立场"**：去味不改作者的观点、结论、数字。如果用户要求改的是这些，不是表达方式，这不是本 skill 的活。
- **"帮我鉴定这一段是不是某个模型生成的"**：这是来源取证，不是风格清洗。本 skill 不能鉴定来源。
- **学术论文/法律文书/正式公文整体润色**：这类场景"中性即正确"，去味只能删协作腔和假互动，不能动太多。事先告诉用户力度很小，问是否仍要继续。

## 边界

- **这是去 AI 味，不是鉴定 AI 来源**。不能说"这段一定是某模型生成的"——这是风格清洗，不是来源取证。
- **不承诺通过检测器**。AI 检测工具本身不准（OpenAI 自己的 classifier 2023 年就下线了）。去味的目的是让文本读起来像人写的，不是骗检测器。
- **不编造"人味"**。不为了像人写的就硬塞个人经历、轶事、笑话、感官细节——那只是另一种表演。
- **不改变事实和立场**。去味不改作者的观点、结论、数据，只改表达方式。
- **场景决定力度**。技术文档不需要"人味"，中性就是对的；别给 API 文档注入灵魂。
