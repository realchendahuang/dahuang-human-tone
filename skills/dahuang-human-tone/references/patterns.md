# AI 味完整清单

> 中文文本 AI 味的分层清单。词汇层最浅，交流层最露骨，结构层最要命。扫的时候按层扫，改的时候按层改。单个口癖没有判断力，必须结合多个层面和具体场景（见 `guardrails.md`）。
>
> 深层句式破绽（分析师讲解语姿、戏剧化揭露、极值判断、伪学术政论腔等）见 `deep-tells.md`，不在这里重复。

## 一、词汇层

### 1. AI 高频词

多类模型输出里反复出现的词汇模式。它们只能作为结合场景复查的线索，不能单独定性：

深入探讨 / 至关重要 / 关键性的 / 不断演变的格局 / 彰显 / 凸显 / 奠定基础 / 深深植根于 / 不可磨灭 / 赋能 / 助力 / 核心 / 关键 / 深耕 / 沉淀 / 闭环 / 抓手 / 体系化 / 模式化 / 驱动 / 构建 / 打造 / 践行 / 价值感 / 长期主义 / 意义深远 / 前景广阔

**改法**：不是换同义词，是改结构让这个词不再需要存在。"深入探讨 X" → 直接讲 X。"赋能开发者" → "让开发者能……"。"打造生态" → 删掉或换成具体做了什么。

### 2. 路标词 / 讲义腔路标词

AI 在正文里给自己当导游的词，像在念 PPT。没有信息量，删掉不影响意思：

**开场路标**：值得注意的是 / 值得一提的是 / 需要指出的是 / 不可否认的是 / 不难发现 / 不容忽视 / 众所周知 / 不得不说 / 诚然 / 具体来说 / 更重要的是 / 在当今（时代/社会/环境下） / 随着……的不断发展 / 在这个……的时代

**中场路标**：接下来我们 / 下面我们 / 我们可以看到 / 在这个过程中 / 这背后其实 / 由此可见 / 换句话说 / 简而言之 / 归根结底 / 不言而喻 / 可以说 / 某种程度上 / 从某种意义上说 / 本质上 / 核心在于 / 关键在于 / 由此可以看出

**收尾路标**：综上所述 / 总而言之 / 总的来说 / 总体来看 / 一句话总结 / 结论先说

**讲义动作词**（像老师在念讲义）：拆一拆 / 拆开来看 / 盘一盘 / 捋一捋 / 聊一聊 / 划重点 / 敲黑板 / 说白了 / 归根结底 / 简单来说 / 换个角度看 / 不妨这样理解

**改法**：删掉，直接说那件事。"值得注意的是，X 在增长" → "X 在增长"。"下面我们来看 Y" → 直接讲 Y。

### 3. 伪口语化高频词

AI 装作在"跟你聊天"的高频词，实际是表演口语化：

**动作词**：拆解 / 梳理 / 剖析 / 解构 / 聚焦 / 洞察 / 盘点 / 把握 / 厘清 / 理清 / 搞清楚

> "拆解""梳理""剖析"三词几乎是 AI 中文分析类文本的签名词——人类写作者不会这样高频地用。

**亲昵语气**：说白了 / 本质上 / 归根结底 / 一句话概括 / 简单来说

**改法**：删掉动作词，直接讲内容。"我们来拆解一下这个问题" → 直接分析问题。"说白了就是……" → 直接说是什么。

### 4. 伪坦率 / 伪严谨评价词

AI 装作"我跟你说真话""我很严谨"的评价词，实际是表演真诚和严谨：

**伪坦率**：直白 / 坦白 / 坦率 / 无遮拦 / 明白无误 / 毫不含糊 / 赤裸 / 赤裸裸 / 不加掩饰 / 不容回避
- 典型句式：`X 说得很直白` / `答案直白得多` / `数据说得更直白`

**伪严谨**：严谨 / 严密 / 严苛 / 严格 / 精准 / 审慎 / 克制 / 沉稳 / 从容
- 典型句式：`严谨地探讨` / `严密的论证` / `精准地切中` / `审慎地权衡`

**改法**：删掉评价词，让内容自己说话。真坦率不需要说自己坦率。

### 5. 互联网黑话 / 商业空话

AI 特别爱堆的"互联网方法论语"：

赋能 / 助力 / 打造 / 抓手 / 闭环 / 颗粒度 / 对齐 / 拉齐 / 沉淀 / 痛点 / 场景化 / 降本增效 / 底层逻辑 / 顶层设计 / 体感 / 心智 / 链路 / 触达 / 透传 / 拉通

**改法**：换成具体做了什么。"对齐颗粒度" → "统一标准"。"打通链路" → "把流程跑通"。技术语境里如果是真行话可留，看宾语和语境判（见 `guardrails.md`）。

### 6. 渲染性强调词

AI 夸大重要性的词，给普通事实加戏：

深刻的（影响/意义/变革） / 深远的（影响/意义） / 不可磨灭的（贡献/印记） / 毋庸置疑 / 至关重要 / 举足轻重 / 令人瞩目 / 令人惊叹 / 意义非凡 / 前所未有 / 史无前例 / 毫不夸张地说 / 值得深思 / 令人深思 / 引发思考 / 发人深省 / 不禁让人（想到/感叹） / 具有重要意义 / 发挥着关键作用 / 颠覆性 / 范式转移

**改法**：删掉渲染词，留事实。"具有深远意义" → 删掉或换成具体意义是什么。

### 7. 自媒体 / 小红书 AI 腔

AI 模仿自媒体的表演腔（注意：人类也会使用，只有多种模式叠加且不符合场景时才值得处理）：

保姆级 / 硬核 / 干货 / 拆解 / 梳理 / 盘点 / 避坑 / 踩坑 / 一文读懂 / 万字长文 / 建议收藏 / 强烈推荐 / 划重点 / 绝绝子 / 谁懂啊 / 真的会谢 / 姐妹们 / 狠狠（XX 了）

**改法**：按场景判。小红书场景里少量用是地道的，堆叠就是 AI。如果是技术文/公号文却出现这些，直接删。

### 8. 无源引用词

AI 归因给模糊权威而不提供具体来源：

研究表明 / 数据显示 / 有专家指出 / 业内人士认为 / 据报道 / 业界普遍认为 / 观察者指出 / 一些批评者认为 / 多个来源显示

**改法**：删掉模糊归因，直接给结论；如果有具体来源就换成具体引用。"专家认为 X" → "X" 或 "中科院 2019 年调查显示 X"。

### 9. 过渡废话词

AI 段落之间填充的连接词，删掉不影响逻辑：

然而 / 此外 / 与此同时 / 不仅……而且 / 一方面……另一方面 / 尽管如此 / 事实上 / 实际上 / 在此基础上 / 进一步地 / 恰恰 / 正是 / 无疑 / 不外乎

**改法**：删掉，用内容本身的逻辑连接。两个段落如果内容有逻辑关系，不需要"此外"来连。

### 新模型高频口癖（2024-2025）

新一代模型（GPT-4o / Claude 3.5+ / Gemini 2.0+ / 豆包 / Kimi）产生的新签名词，和老口癖共存但更隐蔽：

**结论先行腔**：先说结论 / 一句话总结 / 结论先说 / 划重点 / 敲黑板 / 省流版 / TL;DR

**伪通俗化**：说白了就是 / 说人话就是 / 用人话说就是 / 翻译成大白话就是 / 简单粗暴地说

**话题框架壳**：关于 X 这件事 / X 这个事情 / X 这件事的核心 / 聊聊 X 这件事

**思维方法论腔**：底层逻辑 / 第一性原理 / 思维模型 / 认知升级 / 认知框架 / 心智模型 / 复利思维 / 长期主义

**伪对话感过渡**：有意思的是 / 有趣的是 / 你可能会问 / 这里有个反直觉的点 / 你猜怎么着

**改法**：结论先行腔——删掉"先说结论"，直接说结论。伪通俗化——删掉"说白了"，直接说。话题框架壳——删掉"关于 X 这件事"，直接讲 X。思维方法论腔——换成具体在说什么，不要用一个框架词打包。伪对话感过渡——删掉过渡词，直接讲那个"有意思"的点。

## 二、句式层

### 10. 二分对照壳（最常见，最要命）

AI 最爱的升华句式，把复杂事物硬塞进二元对比：

- `不是 A，而是 B`
- `并非 A，而是 B`
- `不在于 A，而在于 B`
- `不只是 A，更是 B`
- `与其说 A，不如说 B`
- `表面上是 A，本质上是 B`

**改前**：这不是效率问题，而是系统问题。
**改后**：问题出在系统上，效率只是最先暴露出来的部分。

**改法**：把 B 直接说清楚。A 如果多余就删，如果有信息就换普通句式说。不对称对照修法：`A 确实……，但……` 或 `A 是有的，只是……`。

> 这是常见的跨风格模板，但自然写作也会使用对照。只有重复出现、掩盖真实关系或与作者声纹不符时才改。

### 11. 机械先后顺序壳

AI 装作在带读者一步步走的顺序壳：

- `先 A，再 B`
- `先 A，然后 B`
- `第一步……第二步……`
- `从 A 到 B`
- `首先……其次……最后……`
- `一方面……另一方面……`

**改法**：如果是真步骤（教程/流程）可留；如果是分析文的顺序表演，删掉顺序词，用内容本身的递进关系连接。

### 12. 本质拔高壳

AI 给普通观察加"本质""底层"的深度感：

- `真正重要的是`
- `真正决定 X 的是`
- `真正 X 的地方在于`
- `本质上`
- `核心在于`
- `底层逻辑`
- `最后比拼的是……`

**改前**：真正决定成败的不是技术，而是执行力。
**改后**：执行力比技术更影响成败。（或直接删掉这种废话）

**改法**：删掉"真正""本质""核心"，直接说结论。

### 13. 否定式排比

"不仅……而且……" / "这不仅仅是 X，更是 Y" / "不只是 X，更是 Y"

**改前**：这不仅仅是一次更新，而是我们思考生产力方式的革命。
**改后**：这次更新改了批处理和快捷键。

**改法**：删掉否定式，直接说它是什么。

### 14. 三段式法则（rule of three）

AI 强行把想法分成三组以显得全面：

- 三个形容词：`无缝、直观、强大`
- 三个短语：`创新、灵感、行业洞察`
- 三个原因 / 三个方面 / 三个层次

**改法**：改为两项或四项。如果真有三个且各有实质内容，可留；如果第三个是凑数的，删掉。

### 15. -ing 肤浅分析（中文等价物）

在句末挂一个现在分词短语增加虚假深度：

- `……，象征着……`
- `……，反映了……`
- `……，彰显了……`
- `……，凸显了……`
- `……，体现了……`
- `……，证明了……`

**改前**：寺庙使用蓝色、绿色和金色，反映了社区与土地的深厚联系。
**改后**：寺庙使用蓝色、绿色和金色。建筑师说这些颜色呼应当地的蓝帽花和海岸。

**改法**：删掉挂在句末的"……着"分词短语，如果真有分析价值就单独成句。

### 16. 关联句式过度标记

AI 用条件句、全称句、因果句做"讲义化"标记，密度过高：

- 条件：`一旦……就` / `只要……就` / `只有……才`
- 全称：`无论……都` / `不管……都`
- 因果：`正是因为……所以` / `正是……才`
- 趋势开段：`随着……的发展/深入/推进`
- 手段讲义化：`通过……来`

**改法**：同一类关联句式反复承担相同功能、形成可预期节奏时再改。把“通过 X 来 Y”压成实际动作；“随着 X 的发展”如果只是背景壳，就直接讲现状。

### 17. 系动词回避

用复杂结构替代简单的"是/有"：

- `作为 X 的证明` → `X 证明了`
- `设有四个空间` → `有四个房间`
- `象征着` → `是`
- `作为/代表/标志着/充当 [一个]` → `是`
- `拥有/设有/提供 [一个]` → `有`

**改法**：换回简单的"是/有"。简单陈述句不是缺点。

### 18. 同义词循环

AI 有重复惩罚机制，导致同一概念轮换同义词——同一段里"主人公/主要角色/中心人物/英雄"轮着用，指同一个人。

**改法**：同一个概念用同一个词，重复没关系。人类写作者不害怕重复。

### 19. 虚假范围

`从 X 到 Y` 结构，但 X 和 Y 不在有意义的尺度上：

**改前**：从大爆炸的奇点到宏伟的宇宙网，从恒星的诞生到暗物质的神秘舞蹈。
**改后**：这本书覆盖了大爆炸、恒星形成和暗物质理论。

**改法**：删掉虚假范围，直接列具体内容。

### 20. 过度限定

层层叠叠的限定词：

**改前**：可以潜在地可能被认为该政策可能会对结果产生一些影响。
**改后**：该政策可能会影响结果。

**改法**：删掉多余的限定词，保留一个就够。

## 三、结构层

### 21. 过度强调意义

任意话题都要拔高到"标志着/代表着/见证了/是……的体现"——夸大重要性：

- `标志着 X 的关键时刻`
- `代表着 X 的转变`
- `见证了 X 的发展`
- `是 X 的体现/证明/提醒`
- `为……做出贡献`
- `为……奠定基础`
- `不可磨灭的印记`
- `深深植根于`

**改法**：删掉拔高陈述，留具体事实。"标志着区域统计演变的关键时刻" → "成立于 1989 年，负责区域统计"。

### 22. 宣传广告式语言

夸张的宣传腔，给普通描述加戏：

充满活力的 / 丰富的（比喻） / 深刻的 / 令人叹为观止 / 必游之地 / 坐落于 / 位于……的中心 / 开创性的 / 著名的 / 迷人的 / 充满魅力

**改前**：坐落在令人叹为观止的区域内，X 是一座充满活力的城镇，拥有丰富的文化遗产。
**改后**：X 是该地区的一座城镇，以每周集市和 18 世纪教堂闻名。

**改法**：删掉形容词堆叠，留具体特征。

### 23. 提纲式"挑战与未来展望"段

公式化的"挑战"段落：

- `尽管 X 面临若干挑战……`
- `尽管存在这些挑战`
- `凭借……继续蓬勃发展`
- `挑战与遗产`
- `未来展望`

**改法**：删掉整个段落，或换成具体挑战和具体举措。"尽管存在挑战，凭借战略位置继续蓬勃发展" → "2022 年启动了排水项目解决洪水问题"。

### 24. 通用积极结论

模糊的乐观结尾：

- `未来看起来光明`
- `激动人心的时代即将到来`
- `继续追求卓越的旅程`
- `向正确方向迈出的重要一步`
- `未来可期`
- `让我们拭目以待`

**改法**：收在具体下一步。"未来可期" → "明年计划开两个新点"。

### 25. 填充短语

没有信息的填充：

- `为了实现这一目标` → 删
- `值得注意的是` → 删
- `在这个时间点` → `现在`
- `系统具有处理的能力` → `系统可以处理`
- `值得注意的是数据显示` → `数据显示`

**改法**：删掉填充，留有信息的部分。

### 26. 段落同构

连续多段结构和功能完全一样——最容易被读者"感觉"到的 AI 味，但最难用词表抓到：

- 连续三段"观点句 + 解释 + 段尾总结"
- 连续多段长度和内部节奏高度接近
- 每段功能相同（都是"讲一个点 + 升华"）

**改法**：按信息关系重组其中一段：可以拆开、合并或删掉重复总结。不要为了制造参差机械改变段落长度，见 `shape-audit.md`。

### 27. 冒号模板

每个列表项以粗体标题 + 冒号开头：

- `**用户体验：**用户体验通过新界面得到显著改善。`
- `**性能：**性能通过优化算法得到增强。`
- `**安全性：**安全性通过端到端加密得到加强。`

**改法**：拆掉冒号模板，写成正常段落。"更新改进了界面，加快了加载时间，添加了加密。"

### 28. 大纲骨架代替文章

多层标题编号代替散文——一、二、三标题 → 1.2.3 编号 → 条列点。整篇都是大纲，没有段落。

**改法**：默认用散文写。一篇最多保留一到两处真正需要并列的清单。把条列点改成段落。

### 29. 粗体四字标签 + 冒号排比清单

`**概念混淆**：…… **价值模糊**：…… **认知偏差**：…… **战略短视**：……`

**改法**：拆掉四字标签，写成正常句子。

### 30. 意义盖章式收尾

每段段尾都盖一个"意义章"：

- `为后世奠定了理论基础`
- `提供了重要的分析框架`
- `具有重要的战略意义`
- `发挥了关键作用`
- `印证了……的重要性`

**改法**：删掉段尾盖章，让段落在具体内容上收住。

### 31. 元论述与导读宣告

AI 在正文里宣告自己要讲什么，像在念目录：

- `本文将探讨……`
- `本节将分析……`
- `下面我把内容分成六个阶段`
- `理解了这一点，我们就能明白……`
- `设想一个极端情况`

**改法**：删掉宣告，直接讲那件事。

### 32. 升华训诫式结尾

中文论说文最典型的 AI 收尾腔——升华、训诫、造金句、呼告。完整清单和改法见 `deep-tells.md` 第九节"升华训诫结尾与金句叠句"。

**改法**：收在具体事实或克制判断。永远不要造一句口号然后在文章里重复它。

## 四、格式层

### 33. 破折号（——）过载

破折号连续承担强调、转折和补充，容易形成制造式“有力”节奏。

**改法**：检查每个破折号是否承担明确功能。连续使用且可以用普通句法更清楚地表达时再换；作者稳定使用或确有节奏作用时保留。

### 34. 粗体滥用

机械地用粗体强调短语、给术语加粗：

**改前**：它融合了 **OKR（目标和关键结果）**、**KPI（关键绩效指标）** 和视觉战略工具。
**改后**：它融合了 OKR、KPI 和视觉战略工具。

**改法**：粗体只用于真正需要强调或便于扫描的内容，不要给所有术语机械加粗。句内排比粗体没有信息层级作用时去掉。

### 35. 内联标题垂直列表

列表项以粗体标题 + 冒号开头——见第 27 条。格式层的处理和结构层一样：拆掉模板。

### 36. emoji 装饰

用 emoji 装饰标题或列表项：

- `🚀 **启动阶段：**`
- `💡 **关键洞察：**`
- `✅ **下一步：**`

**改法**：删掉 emoji（除非社媒场景刻意用）。标题不需要图标。

### 37. 弯引号

混用中英文标点。中文应该用中文引号（""或「」），不要混入英文弯引号（""）。

**改法**：统一成中文标点。

## 五、交流层

### 38. 协作交流痕迹

聊天机器人对话痕迹留在正文里：

- `希望这对您有帮助`
- `当然！` / `一定！`
- `您说得完全正确！`
- `请告诉我`
- `如果您想让我扩展任何部分`
- `如果你有其他问题`

**改法**：出现在独立正文里时通常删除。客服回复、邮件或对话本来就需要交互信息时，只删复述、谄媚和多余邀功，保留有效的下一步。

### 39. 知识截止日期免责声明

- `截至 [日期]`
- `根据我最后的训练更新`
- `虽然具体细节有限……`
- `基于可用信息……`

**改法**：删掉免责声明，直接给信息。信息不确定就说不确定，不要加 AI 的免责标签。

### 40. 谄媚 / 卑躬屈膝语气

过于积极、讨好的语言：

- `好问题！`
- `您说得完全正确`
- `这是一个很好的观点`
- `让我来为您解释`
- `你的观察力太敏锐了`
- `这个思路简直绝了`

**改法**：删掉谄媚开场，直接回应内容。

### 41. 假互动结尾

AI 在文末假装和读者互动：

- `你觉得呢？`
- `是不是很有启发？`
- `你有没有类似经历？`
- `你现在卡在哪一步？`
- `你有什么看法？`

**改法**：删掉，除非用户要求互动。好文章不需要在结尾摇尾乞怜求互动。

### 42. 二人称过密

AI 过度使用"你""大家"，把每个读者都拉进来说话：

- 全文"你可以……""你需要……""你会发现……"
- "大家""你们"

**改法**：看“你”是否真的指向读者并服务场景。聊天、教程和营销可以高频使用；正式文本里如果只是模拟亲密或反复指挥读者，就改成具体主语或直接陈述。

### 43. 过度接住 / 心理判断腔

AI 写"咨询式安抚"，假装在接住读者的情绪：

- `我就在这里`
- `稳稳地接住你`
- `你只是太久没被稳稳接住了`
- `你不是敏感/不是想太多/不是矫情`
- `你太清醒了/太懂了/太对了`
- `这次我懂了，我真的懂了`

**改法**：看宾语判（见 `guardrails.md`）。宾语是"你/情绪/脆弱" = AI 表演腔，删。宾语是"流量/请求/峰值/异常" = 技术语境的行话，留。

### 44. AI 主动出击腔

AI 在回复里过度主动，像在邀功：

- `我已确认`
- `我立马开始`
- `要不要我……`
- `如果你愿意……`
- `你一回复我就……`
- `顺手`
- `我先……`

**改法**：删掉主动出击的铺垫，直接做或者直接说。

### 45. 翻译腔

中文特有 AI 味——AI 的中文读起来像从英文翻译过来的：

- `一个……的……的……` 长定语结构
- 被动语态堆砌：`被优化` / `被改进` / `被赋予`
- `基于……` 开头
- `通过……来……`
- `对于……而言`
- `在……方面`
- `从……的角度来看`

**改法**：换成地道中文。长定语拆成短句。“通过 X 来 Y”改成直接说 Y。“对于 X 而言”改成“对 X 来说”或直接删。只有责任主体明确且主动句更清楚时才改被动；学术、法律、未知执行者和结果导向场景可以保留。

## 六、跨层组合模式

这些模式常同时涉及结构、句式和交流方式。不要和前面的单点重复计数。

### 46. 权威揭示壳

“真正的问题是” / “归根结底” / “说到底” / “更深层的原因” / “核心真相”

**问题**：先摆出“我来揭示真相”的姿态，后面只重复普通判断。

**改法**：删掉揭示仪式，直接说具体判断和条件。

### 47. 宣布自己将要做什么

“让我们深入探讨” / “下面拆解” / “接下来看看” / “你需要知道的是”

**问题**：正文像讲义脚本，先播报动作再执行。

**改法**：直接进入内容。教学场景确实需要导航时，保留最短的结构提示。

### 48. 标题后复述标题

标题下面先来一句同义暖场，再进入真正内容。

**改法**：删掉暖场句，让标题直接接事实、论证或步骤。

### 49. Diff-anchored 写作

“本次新增了” / “这次改为” / “此前采用” / “为了替代旧方案”

**问题**：普通文档像在讲本次提交，而不是描述当前事实。

**改法**：直接写现在如何工作。Changelog、迁移指南和版本复盘不改。

### 50. 制造式短句和金句堆叠

连续多句都试图“落地”：

“规则变了。旧时代结束了。没有退路。”

**问题**：短句不是为了节奏，而是为了制造可截图的戏剧感。

**改法**：合并成一个具体判断，保留真正需要强调的一句。

### 51. 警句公式

“X 是 Y 的货币” / “X 是一面镜子” / “X 不是工具，而是答案” / “X 成了陷阱”

**问题**：抽象比喻听起来深刻，却没有增加可验证信息。

**改法**：说清楚对象、机制和实际后果。比喻确实帮助理解时保留，不要再解释一遍。

### 52. 假坦率开场

“说实话？” / “讲真” / “朋友们” / “问题来了” / “你猜怎么着”

**问题**：先表演亲近或坦率，再说普通内容。

**改法**：直接说那件事。作者样本里稳定使用、且符合平台语感时保留。

## 速查：最常出现的 10 个口癖

改完检查这 10 个有没有残留，基本就能去掉八成 AI 味：

1. `不是……而是` （二分对照壳）
2. `值得注意的是` （路标词）
3. `深入探讨` / `拆解` / `梳理` / `剖析` （伪口语化动作词）
4. `本质上` / `核心在于` / `底层逻辑` （本质拔高壳）
5. `赋能` / `助力` / `打造` （互联网黑话）
6. `综上所述` / `总的来说` / `先说结论` （收尾路标 + 结论先行腔）
7. `希望这能帮到你` / `你觉得呢？` （协作痕迹 + 假互动）
8. `——` 破折号过载
9. `**术语**` 粗体滥用
10. 每段都差不多长（段落同构）
