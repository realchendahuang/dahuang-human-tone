# Changelog

本项目使用语义化版本号记录 Skill 的重要变化。

## 3.0.0 — 2026-07-10

- 将去味流程重构为“编辑合同 → 内容账本 → 五遍编辑引擎 → 验证回滚”。
- 增加轻改、标准和深改三档强度，允许自然文本不改。
- 增加技术、学术、法律、营销、社媒、公开写作和叙事体裁路由。
- 增加作者样本声纹校准，只迁移风格，不复制样本内容。
- 保护学术 hedging、非母语作者表达、营销 CTA、技术术语、叙事伏笔和人物声口。
- 合并重叠诊断，增加“需复核”和停止条件，避免为了展示能力过度改写。
- 增加跨体裁回归用例和 2026 Humanizer Skill 生态调研。
- 清理固定句长、词频、标点禁令和一刀切主动语态等伪量化规则。
- 重写示例，确保改写不新增事实、经历、配置、来源或作者立场。

## 2.0.0 — 2026-07-10

- 重写 Skill 入口，按改写、诊断和轻改三种模式执行。
- 改成按场景懒加载 reference，不再要求所有文本执行形状量化。
- 删除没有可靠依据的固定句长、段长和词频阈值。
- 把“AI 判定”收紧为可观察的写作诊断，明确禁止来源归因和伪精确评分。
- 将事实保护从“一字不改”修正为“含义与身份不变”，减少对正常句法调整的阻碍。
- 增加长 reference 的目录，并清理法证式“定罪”措辞。
- 对齐 Codex UI 元数据，只保留受支持字段。
